# STUDENT MODULE - COMPLETE SCHOOL MANAGEMENT SYSTEM
## Comprehensive Development Prompt

---

## PROJECT OVERVIEW
Build a complete student-facing module for the School Management System with all features adapted from the teacher module. The student interface should be intuitive, mobile-friendly, and provide students with easy access to their academic information, homework, attendance, notices, and grades.

---

## DESIGN SYSTEM

### Color Palette
- **Primary Blue**: #1897C6 (Main actions, links, primary buttons)
- **Accent Orange**: #D88931 (Highlights, warnings, pending items)
- **Dark Orange**: #D87331 (Secondary actions)
- **Teal/Cyan**: #67BAC3 (Success states, completed items)
- **Neutral Gray**: #535359 (Text primary)
- **Light Gray**: #6B7280 (Text secondary)
- **Background**: #F9FAFB (Page background)
- **Card Background**: #FFFFFF
- **Border**: #E5E7EB

### Typography
- **Font Family**: Use Geist Sans for body text and Geist Mono for code/numbers
- **Headings**: Bold, 2xl to 3xl font size
- **Body Text**: Regular, base to sm font size with leading-relaxed
- **Labels**: Semibold, sm font size

### Layout Principles
- **Mobile-First**: Responsive design starting from mobile
- **Card-Based**: Use elevated cards with rounded corners (rounded-xl, rounded-2xl)
- **Gradient Accents**: Subtle gradients for headers and important sections
- **Border Highlights**: Left or top borders (border-l-4) for visual hierarchy
- **Spacing**: Consistent gap-4, gap-6, p-6, mb-8 spacing

## CORE FEATURES & PAGES

### 1. STUDENT DASHBOARD (Home Page)
**Route**: `/app/page.tsx` or `/app/dashboard/page.tsx`

**Layout Structure**:
- **Header Section**: Welcome message with student name, profile picture, class, and roll number
- **Quick Stats Cards** (4 cards in grid):
  - Pending Homework (count with orange accent)
  - Attendance Percentage (with green/red indicator)
  - Upcoming Classes (today's schedule count)
  - Unread Notices (count with red badge)

**Today's Schedule Section**:
- List of classes for the current day
- Display: Subject name, Time, Teacher name, Room number
- Visual indicators for current/upcoming/completed classes
- Color coding: Current class (blue highlight), Upcoming (gray), Completed (green checkmark)

**Recent Activity Section**:
- Last 5 homework assignments (title, subject, due date, status)
- Recent notices (truncated with "Read more")
- Recent grade updates

**Quick Actions**:
- View All Homework
- Check Attendance
- View Notices
- View Grades

### 2. HOMEWORK PAGE
**Route**: `/app/homework/page.tsx`

**View Sections**:

**Active Homework Tab**:
- Display all pending homework assignments
- Each card shows:
  - Subject and Class
  - Assignment Title
  - Description/Instructions
  - Due Date with countdown (X days remaining)
  - Attachments from teacher (images, videos, PDFs) with download/view
  - Submission Status: Not Submitted / Submitted / Late Submission
  - Grade (if graded): Display grade with color coding

**Homework Card Actions**:
- **Submit Work Button**: Opens submission modal
- **View Details**: Expand to see full instructions
- **Download Attachments**: Download teacher's reference materials

**Submission Modal**:
- Upload files (images, videos, PDFs, documents)
- Text area for written answers/notes
- Multiple file upload support with preview
- Remove uploaded files before submit
- Submit button with confirmation
- Show submission timestamp after submit

**Submitted Homework Tab**:
- List of all submitted homework
- Show: Subject, Title, Submitted Date, Grade, Teacher Feedback
- Option to view submitted work
- Download teacher's feedback/comments if any

**Graded Homework Tab**:
- Filter by subject
- Show grade statistics (average, highest, lowest)
- Visual charts showing grade trends

## TECHNICAL SPECIFICATIONS

### Technology Stack
- **Framework**: Next.js 16 with App Router
- **Language**: TypeScript
- **Styling**: Tailwind CSS v4
- **UI Components**: shadcn/ui components
- **Icons**: lucide-react
- **State Management**: React hooks (useState, useEffect)
- **Data Fetching**: SWR for client-side data (if needed)

### Database Integration
**Required**: Use Supabase or Neon PostgreSQL for backend

**Database Tables** (Student Module):
1. **students** - Student information
2. **homework_submissions** - Student homework submissions
3. **attendance_records** - Daily attendance data
4. **grades** - Exam/test grades
5. **notices_read** - Track which notices student has read
6. **class_schedule** - Timetable data
7. **notifications** - Student notifications

**Authentication**:
- Student login (email/roll number + password)
- Secure session management
- Parent access (separate login for parents to view student data)

### File Structure
```
/app
```

## SAMPLE DATA STRUCTURE

### Student Object
```typescript
interface Student {
  id: string;
  name: string;
  rollNumber: string;
  class: string;
  section: string;
  email: string;
  phone: string;
  profilePicture?: string;
  dateOfBirth: string;
  bloodGroup: string;
  address: string;
  parentName: string;
  parentPhone: string;
  parentEmail: string;
  admissionNumber: string;
  academicYear: string;
}
```

### Homework Submission Object
```typescript
interface HomeworkSubmission {
  id: string;
  homeworkId: string;
  studentId: string;
  submittedDate: string;
  files: { name: string; url: string; type: string }[];
  textContent?: string;
  status: 'submitted' | 'late' | 'graded';
  grade?: string;
  teacherFeedback?: string;
  marks?: number;
}
```

### Attendance Record Object
```typescript
interface AttendanceRecord {
  id: string;
  studentId: string;
  date: string;
  subject: string;
  status: 'present' | 'absent' | 'leave' | 'holiday';
  markedBy: string;
  remarks?: string;
}
```
